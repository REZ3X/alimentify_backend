Authentication

Get Google Auth URL

GET
/api/auth/google

Initiates Google OAuth 2.0 authentication flow.

Response:

{
  "auth_url": "https://accounts.google.com/o/oauth2/v2/auth?..."
}
Public endpoint - No authentication required

Google OAuth Callback

GET
/api/auth/google/callback

Handles OAuth callback and creates/logs in user.

Query Parameters:

code OAuth authorization code from Google

Response:

// For new users:
Redirect to: {FRONTEND_URL}/auth/check-email?email={email}

// For existing verified users:
Redirect to: {FRONTEND_URL}/?token={jwt_token}

Verify Email

GET
/api/auth/verify-email

Verifies user email and returns JWT token for auto-login.

Query Parameters:

token Email verification token

Response:

{
  "success": true,
  "message": "Email verified successfully",
  "token": "jwt_token_here",
  "user": {
    "id": "user_id",
    "email": "user@example.com",
    "name": "User Name"
  }
}

Get Current User

GET
/api/auth/me

Retrieves authenticated user information.

Headers:

Authorization: Bearer {jwt_token}

Response:

{
  "id": "user_id",
  "email": "user@example.com",
  "name": "User Name",
  "picture": "profile_picture_url"
}
Protected endpoint - Requires authentication

Logout

POST
/api/auth/logout

Logs out the current user and invalidates session.

Headers:

Authorization: Bearer {jwt_token}

Response:

{
  "success": true,
  "message": "Logged out successfully"
}
Protected endpoint - Requires authentication

Health Profile

Create or Update Profile

POST
/api/health/profile

Creates or updates user health profile and calculates daily nutrition targets.

Headers:

Authorization: Bearer {jwt_token}
Content-Type: application/json

Request Body: 

{
  "age": 25,
  "gender": "male",
  "height_cm": 175,
  "weight_kg": 70,
  "activity_level": "moderately_active",
  "goal": "maintain_weight"
}

Activity Levels:

sedentary - Little or no exercise
lightly_active - Light exercise 1-3 days/week
moderately_active - Moderate exercise 3-5 days/week
very_active - Hard exercise 6-7 days/week
extremely_active - Very hard exercise & physical job

Goals:

lose_weight - 500 calorie deficit
maintain_weight - Maintenance calories
gain_weight - 500 calorie surplus

Response:

{
  "success": true,
  "profile": {
    "age": 25,
    "gender": "male",
    "height_cm": 175,
    "weight_kg": 70,
    "activity_level": "moderately_active",
    "goal": "maintain_weight",
    "daily_calories": 2500,
    "daily_protein_g": 140,
    "daily_carbs_g": 281,
    "daily_fat_g": 83
  }
}
Protected endpoint - Requires authentication

Meal Tracking

Log Meal

POST
/api/meals/log

Logs a meal entry with nutritional information.

Headers:

Authorization: Bearer {jwt_token}
Content-Type: application/json

Request Body: 

{
  "meal_type": "breakfast",
  "food_name": "Scrambled Eggs",
  "calories": 250,
  "protein_g": 18,
  "carbs_g": 2,
  "fat_g": 20,
  "serving_size": "2 eggs",
  "notes": "With butter"
}

Meal Types:

breakfast, lunch, dinner, snack

Response:

{
  "success": true,
  "meal": {
    "id": "meal_id",
    "user_id": "user_id",
    "meal_type": "breakfast",
    "food_name": "Scrambled Eggs",
    "calories": 250,
    "protein_g": 18,
    "carbs_g": 2,
    "fat_g": 20,
    "serving_size": "2 eggs",
    "notes": "With butter",
    "created_at": "2025-01-01T08:00:00Z"
  }
}
Protected endpoint - Requires authentication

Get Daily Meals

GET
/api/meals/daily?date=2025-01-01

Retrieves all meals for a specific date with daily totals.

Query Parameters:

date Date in YYYY-MM-DD format (optional, defaults to today)

    "calories_remaining": 650,
    "protein_remaining_g": 20,
    "carbs_remaining_g": 101,

AI Nutrition Analysis

Analyze Food Image

POST
/api/nutrition/analyze

Analyzes food image using Google Gemini AI to extract nutritional information.

Headers:

Authorization: Bearer {jwt_token}
Content-Type: multipart/form-data

Form Data:

image Image file (max 10MB, JPEG/PNG)

Response:

{
  "success": true,
  "analysis": "{\n  \"food_name\": \"Cheeseburger\",\n  \"serving_size\": \"1 burger\",\n  \"calories\": \"600-700\",\n  \"macronutrients\": {\n    \"protein\": \"30-35g\",\n    \"carbohydrates\": \"45-50g\",\n    \"fat\": \"35-40g\"\n  },\n  \"health_score\": 5,\n  \"health_notes\": \"...\",\n  \"recommendations\": \"...\"\n}",
  "timestamp": "2025-01-01T12:00:00Z"
}
Protected endpoint - Requires authentication

Analyze Food from Text

POST
/api/nutrition/analyze-text

Analyzes food description and provides nutrition information.

Request Body: 

{
  "food_description": "200g grilled chicken breast"
}

Response:

{
  "food_name": "Grilled Chicken Breast",
  "calories": 330,
  "protein_g": 62,
  "carbs_g": 0,
  "fat_g": 7,
  "serving_size": "200g"
}
Protected endpoint - Requires authentication

    "serving_size_g": 100,
    "protein_g": 31,
    "carbohydrates_total_g": 0,
    "fat_total_g": 3.6

Food Wiki (USDA FoodData Central)

Search Foods

GET
/api/food-wiki/search?query=apple&pageSize=10

Searches USDA FoodData Central database.

Query Parameters:

query Search term (required)
pageSize Results per page (default: 25, max: 200)
pageNumber Page number (default: 1)
dataType Foundation, SR Legacy, Survey (optional)
Protected endpoint - Requires authentication

Get Food Details

GET
/api/food-wiki/{fdc_id}

Retrieves detailed information for a specific food item.

Protected endpoint - Requires authentication

Get Multiple Foods

POST
/api/food-wiki/foods

Retrieves details for multiple food items by their FDC IDs.

Request Body: 

{
  "fdcIds": [534358, 747447, 787782]
}
Protected endpoint - Requires authentication

Recipes (TheMealDB)

Search Recipes

GET
/api/recipes/search?query=chicken

Searches recipes by name.

Query Parameters:

query Recipe name or ingredient
Protected endpoint - Requires authentication

Get Random Recipes

GET
/api/recipes/random?count=6

Retrieves random recipes.

Query Parameters:

count Number of recipes (default: 1)
Protected endpoint - Requires authentication

Get Recipe by ID

GET
/api/recipes/{meal_id}

Retrieves detailed recipe information by ID.

Protected endpoint - Requires authentication

Filter by Category

GET
/api/recipes/category/{category}

Filters recipes by category (e.g., Seafood, Vegetarian, Dessert).

Protected endpoint - Requires authentication

Filter by Area

GET
/api/recipes/area/{area}

Filters recipes by cuisine/area (e.g., Italian, Chinese, Mexican).

Protected endpoint - Requires authentication
